# Support / Service Tickets Module

## Overview

The Support / Service Tickets module enables efficient tracking and management of customer service requests, technical issues, and support activities across all projects. It provides a centralized system for handling customer inquiries, tracking resolution progress, and maintaining a complete history of support interactions.

## Purpose of Support Tickets

### What Are Support Tickets?

Support tickets are customer service requests linked to specific projects. They serve as a structured way to track, manage, and resolve customer issues, technical problems, and service requests throughout the project lifecycle.

### Why Use Support Tickets?

**Structured Tracking**:
- Organize customer service requests systematically
- Track issues from creation to resolution
- Maintain complete history of support interactions
- Ensure no customer issue is overlooked

**Improved Communication**:
- Document customer concerns clearly
- Track follow-up activities and dates
- Maintain communication history
- Enable team collaboration on complex issues

**Performance Monitoring**:
- Monitor ticket resolution times
- Track overdue follow-ups
- Analyze support workload
- Measure customer service effectiveness

**Accountability**:
- Assign responsibility for ticket resolution
- Track who handled each ticket
- Document resolution steps
- Maintain audit trail

### When to Create a Ticket

**Customer Issues**:
- Customer reports a problem
- Technical issue requires investigation
- Service request needs tracking
- Customer inquiry needs follow-up

**Project-Related Issues**:
- Installation problems
- Equipment malfunctions
- Compliance issues
- Documentation requests

**Follow-Up Required**:
- Customer needs callback
- Issue requires monitoring
- Resolution needs verification
- Ongoing support needed

## Ticket Number Format

### Understanding Ticket Numbers

Every support ticket is assigned a unique ticket number when created. This number serves as the primary identifier for the ticket throughout its lifecycle.

### Format Details

**Format**: RE + 8 digits

**Examples**:
- RE12345678
- RE87654321
- RE00001234

**Components**:
- **RE**: Prefix indicating "Rayenna Energy" or "Request/Enquiry"
- **8 digits**: Sequential number (padded with zeros if needed)

### Ticket Number Generation

**Automatic Generation**:
- Ticket number is automatically generated by the system
- Assigned when ticket is created
- Cannot be changed after creation
- Ensures uniqueness across all tickets

**Uniqueness**:
- Each ticket has a unique number
- System prevents duplicate numbers
- Numbers are sequential
- Easy to reference in communications

### Using Ticket Numbers

**Reference in Communication**:
- Use ticket number when discussing issues with customers
- Include in email communications
- Reference in phone conversations
- Use for tracking and follow-up

**Search and Filter**:
- Search for tickets by number
- Reference tickets by number
- Use in reports and documentation
- Track ticket history

## Creating Tickets from Projects

### Who Can Create Tickets

**Authorized Roles**:
- **Sales users**: Can create tickets for their projects
- **Operations users**: Can create tickets for any project
- **Administrators**: Can create tickets for any project

**Permissions**:
- Sales users can create tickets for projects assigned to them
- Operations users have broader access
- Admin users have full access

### Creating a Ticket from Project Page

**Step 1: Access Project**
1. Go to **Projects** page
2. Find and open the project
3. Scroll to **Support / Service Tickets** section

**Step 2: Create Ticket**
1. Click **Create Ticket** button
2. Ticket creation modal opens

**Step 3: Enter Ticket Information**

**Required Fields**:
- **Title**: Brief description of the issue or request
  - Be specific and clear
  - Example: "Solar panel not generating power"
  - Example: "Customer needs installation date update"

**Optional Fields**:
- **Description**: Detailed information about the issue
  - Provide context and background
  - Include relevant details
  - Note any previous communication
  - Add technical details if applicable

**Step 4: Save Ticket**
1. Review information for accuracy
2. Click **Create** button
3. Ticket is created with unique ticket number
4. Ticket appears in project's ticket list
5. Success message confirms creation

### Ticket Information Captured

**Automatic Information**:
- **Ticket Number**: Automatically generated (RE + 8 digits)
- **Project**: Linked to the project from which it was created
- **Customer**: Linked to project's customer
- **Created Date**: Current date and time
- **Created By**: User who created the ticket
- **Status**: Automatically set to "OPEN"

**User-Provided Information**:
- **Title**: Issue description
- **Description**: Detailed information (optional)

### Creating Tickets Best Practices

**Clear Titles**:
- Use descriptive, specific titles
- Include key information in title
- Make titles searchable
- Avoid vague descriptions

**Complete Descriptions**:
- Provide sufficient detail
- Include relevant context
- Note customer contact information
- Document any previous attempts

**Timely Creation**:
- Create tickets promptly when issues arise
- Don't delay ticket creation
- Capture information while fresh
- Ensure nothing is missed

## Follow-ups with Dates

### Understanding Follow-ups

Follow-ups are activity notes added to tickets to track progress, document communications, and schedule future actions. Each follow-up can include notes and an optional follow-up date.

### Adding Follow-ups

**Who Can Add Follow-ups**:
- **Sales users**: Can add follow-ups to tickets
- **Operations users**: Can add follow-ups to tickets
- **Administrators**: Can add follow-ups to tickets

**How to Add Follow-up**:

**From Project Page**:
1. Go to project's Support / Service Tickets section
2. Click **View Ticket** for the ticket
3. Scroll to "Add Follow-up" section
4. Enter follow-up information

**From Support Tickets Dashboard**:
1. Go to Support Tickets Dashboard
2. Click on ticket number or **View** button
3. Ticket detail drawer opens
4. Scroll to "Add Follow-up" section

**Follow-up Form**:
- **Note** (Required): Activity description or communication details
  - Document what was done
  - Record customer communication
  - Note progress made
  - Include relevant information

- **Follow-up Date** (Optional): Date for next action or follow-up
  - Set when next action is needed
  - Schedule customer callback
  - Plan resolution verification
  - Track pending activities

**Saving Follow-up**:
1. Enter note (required)
2. Optionally set follow-up date
3. Click **Add Follow-up** button
4. Follow-up is added to ticket
5. Activity appears in timeline

### Follow-up Timeline

**Activity History**:
- All follow-ups displayed chronologically
- Shows date and time of each activity
- Displays who added each follow-up
- Includes follow-up dates if set

**Timeline View**:
- Most recent activities at top
- Complete history visible
- Easy to track progress
- Clear communication trail

### Follow-up Dates

**Purpose**:

**Using Follow-up Dates**:

**Overdue Follow-ups**:

## Resolution and Closure

### Understanding Ticket Resolution

Ticket resolution involves addressing the customer's issue, verifying the solution, and closing the ticket when the matter is resolved.

### Ticket Status

**Status Types**:
- **OPEN**: New ticket, not yet addressed
- **IN_PROGRESS**: Ticket is being worked on
- **CLOSED**: Ticket resolved and closed

**Status Progression**:
- Tickets start as OPEN
- Move to IN_PROGRESS when work begins
- Close when issue is resolved

### Closing Tickets

**Who Can Close Tickets**:
- **Sales users**: Can close tickets
- **Operations users**: Can close tickets
- **Administrators**: Can close tickets

**How to Close Ticket**:

**From Project Page**:
1. Go to project's Support / Service Tickets section
2. Click **View Ticket** for the ticket
3. Click **Close Ticket** button
4. Confirm closure
5. Ticket status changes to CLOSED

**From Support Tickets Dashboard**:
1. Open ticket detail drawer
2. Click **Close Ticket** button
3. Confirm closure
4. Ticket status changes to CLOSED

**Closure Information**:
- **Closed Date**: Automatically recorded when ticket is closed
- **Closed By**: User who closed the ticket
- **Status**: Changes to CLOSED
- **History**: Closure is recorded in activity timeline

### Resolution Process

**Step 1: Address Issue**:
- Investigate the problem
- Identify root cause
- Develop solution
- Implement fix

**Step 2: Verify Resolution**:
- Test the solution
- Confirm issue is resolved
- Verify with customer if needed
- Document verification

**Step 3: Close Ticket**:
- Add final follow-up note
- Document resolution
- Close the ticket
- Record closure date

### Closed Ticket Restrictions

**Limited Actions**:
- Closed tickets cannot be reopened (except by Admin)
- No new follow-ups can be added (except by Admin)
- Ticket remains in system for history
- Can be viewed but not modified

**Admin Override**:
- Administrators can add follow-ups to closed tickets
- Admin can reopen tickets if needed
- Admin has full access to all tickets

### Resolution Best Practices

**Complete Resolution**:
- Ensure issue is fully resolved
- Verify solution works
- Confirm with customer if needed
- Document resolution steps

**Proper Closure**:
- Add final follow-up before closing
- Document what was done
- Note any follow-up needed
- Close ticket promptly

**Documentation**:
- Maintain complete resolution history
- Document all steps taken
- Note any lessons learned
- Keep records for future reference

## Troubleshooting

### Cannot Create Ticket

**Issue**: "Create Ticket" button not visible or creation fails

**Solutions**:
- Verify you have Sales, Operations, or Admin role
- Check you're on the correct project page
- Ensure project exists and is accessible
- Refresh page and try again
- Contact administrator if issue persists

### Cannot Add Follow-up

**Issue**: Cannot add follow-up to ticket

**Solutions**:
- Verify ticket is not closed (unless Admin)
- Check you have Sales, Operations, or Admin role
- Ensure note field is filled
- Refresh page and try again
- Contact administrator if needed

### Cannot Close Ticket

**Issue**: "Close Ticket" button not visible or closure fails

**Solutions**:
- Check you have Sales, Operations, or Admin role
- Verify ticket is not already closed
- Ensure ticket exists and is accessible
- Refresh page and try again
- Contact administrator if needed

### Dashboard Not Loading

**Issue**: Dashboard shows loading or error

**Solutions**:
- Check internet connection
- Refresh page
- Clear browser cache
- Try different browser
- Contact administrator if issue persists

### Tickets Not Appearing

**Issue**: Expected tickets not showing in dashboard

**Solutions**:
- Check active filters
- Clear all filters
- Verify ticket status
- Check project access permissions
- Refresh page
- Contact administrator if needed
