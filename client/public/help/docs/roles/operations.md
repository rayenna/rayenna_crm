# Operations User Guide

## Welcome

This guide is designed specifically for Operations users of Rayenna Energy CRM. It covers everything you need to know to effectively manage project execution, installations, subsidies, and customer support.

## Your Dashboard

### Overview

Your dashboard shows key operational metrics and helps you prioritize your daily work.

### Key Metrics

**Top Row Metrics**:
- **Pending Installation**: Number of projects waiting for installation to begin
- **Submitted for Subsidy**: Number of projects where subsidy has been submitted
- **Subsidy Credited**: Number of projects where subsidy has been received

**Pending Subsidy List**:
- Shows projects waiting for subsidy credit
- Displays customer name and days pending
- Helps you identify projects needing attention
- Click on projects to view details

### Charts and Visualizations

- **Project Value and Profit by Financial Year**: See operational performance over time
- **Project Value by Segment**: Visual breakdown of projects by type

### Using Dashboard Filters

- Select **Financial Year** to view data for specific periods
- Select **Month** to narrow down to specific months
- Use filters to analyze trends and identify bottlenecks

## Project Execution Tracking

### Understanding Your Project Access

**Projects You Can View**:
- **Confirmed Order**: Customer has approved, ready for installation
- **Installation**: Installation work is in progress
- **Completed**: Installation finished, system operational
- **Completed - Subsidy Credited**: Fully complete including subsidy

**Projects You Cannot View**:
- Lead, Site Survey, Proposal stages (handled by Sales team)
- Lost projects (no longer active)

### Finding Projects to Work On

**From Dashboard**:
- Check "Pending Installation" metric
- Review "Pending Subsidy" list
- Click on projects to open details

**From Projects Page**:
1. Go to **Projects** from top menu
2. Filter by status:
   - Select "Confirmed Order" for new installations
   - Select "Installation" for in-progress work
   - Select "Completed" for finished projects
3. Use search to find specific customers or projects

### Tracking Project Progress

**Project Lifecycle Dates**:
The system tracks important dates throughout project execution. You update these dates as work progresses:

**Registration and Compliance Dates**:
- **MNRE Portal Registration Date**: When project was registered on MNRE portal
- **Feasibility Date (DISCOM)**: Date DISCOM approved feasibility
- **Registration Date (DISCOM)**: Date of DISCOM registration

**Installation Dates**:
- **Installation Completion Date**: When installation work was finished
- **Completion Report Submission Date**: When completion report was submitted

**Subsidy Dates**:
- **Net Meter Installation Date**: When net meter was installed
- **Subsidy Credited Date**: When subsidy amount was received

### Updating Project Lifecycle Dates

**How to Update**:
1. Open the project detail page
2. Click **Edit** button
3. Scroll to **Project Lifecycle** section
4. Enter or update the relevant dates
5. Click **Save**

**When to Update**:
- Update dates as soon as each milestone is completed
- Don't wait - update immediately after completing each step
- Accurate dates help track project progress and identify delays

### Updating Project Status

**Status Progression**:
- **Confirmed Order** → **Installation** (when installation begins)
- **Installation** → **Completed** (when installation finishes)
- **Completed** → **Completed - Subsidy Credited** (when subsidy is received)

**How to Update Status**:
1. Open the project
2. Click **Edit**
3. Find **Project Status** in Sales & Commercial section
4. Select the new status
5. Click **Save**

**Important**: Update status promptly to keep the system current and help other teams track progress.

## Installations

### Installation Workflow

**Step 1: Receive Confirmed Order**
- Project appears in your dashboard as "Pending Installation"
- Status is "Confirmed Order"
- Review project details and customer information

**Step 2: Begin Installation**
- Update project status to "Installation"
- Update relevant dates (MNRE registration, DISCOM dates if applicable)
- Upload any required documents

**Step 3: Complete Installation**
- Update "Installation Completion Date"
- Update "Completion Report Submission Date"
- Upload completion documents and photos
- Change status to "Completed"

**Step 4: Post-Installation**
- Ensure all documents are uploaded
- Verify all dates are entered
- Update status when subsidy is credited

### Managing Installation Dates

**Key Dates to Track**:
- **MNRE Portal Registration**: Register project on MNRE portal
- **DISCOM Feasibility**: Get feasibility approval from DISCOM
- **DISCOM Registration**: Complete DISCOM registration
- **Installation Start**: When installation work begins
- **Installation Completion**: When installation is finished
- **Completion Report**: When report is submitted

**Best Practices**:
- Enter dates as soon as each step is completed
- Don't batch update - enter dates daily
- Verify dates are correct before saving
- Use actual completion dates, not planned dates

### Installation Documentation

**Documents to Upload**:
- Installation photos and videos
- Completion certificates
- Compliance documents
- DISCOM approval documents
- MNRE registration documents
- Net meter installation documents

**When to Upload**:
- Upload documents immediately after receiving them
- Don't wait until project is complete
- Upload as you complete each step

## Subsidy Tracking

### Understanding Subsidy Process

**Subsidy Stages**:
1. **Installation Complete**: System installed and operational
2. **Net Meter Installed**: Net meter installation completed
3. **Subsidy Submitted**: Application submitted for subsidy
4. **Subsidy Credited**: Subsidy amount received

### Tracking Subsidy Status

**From Dashboard**:
- View "Submitted for Subsidy" count
- View "Subsidy Credited" count
- Check "Pending Subsidy" list for projects needing attention

**From Project Detail Page**:
- View subsidy-related dates
- See current subsidy status
- Track days pending for subsidy credit

### Updating Subsidy Information

**Net Meter Installation Date**:
- Update when net meter is installed
- This is required before subsidy can be credited
- Enter the actual installation date

**Subsidy Credited Date**:
- Update when subsidy amount is received
- This is the final step in the subsidy process
- Change project status to "Completed - Subsidy Credited"

**How to Update**:
1. Open the project
2. Click **Edit**
3. Scroll to **Project Lifecycle** section
4. Enter or update subsidy dates
5. Update project status if subsidy is credited
6. Click **Save**

### Monitoring Pending Subsidies

**Dashboard Pending Subsidy List**:
- Shows projects waiting for subsidy credit
- Displays number of days pending
- Helps identify projects that need follow-up
- Click on project to view details

**Best Practices**:
- Check pending subsidy list daily
- Follow up on projects pending more than 30 days
- Update subsidy credited date immediately when received
- Keep subsidy status current

## Support Ticket Handling

### Understanding Support Tickets

Support tickets track customer service requests and issues. As Operations user, you handle tickets for all projects, not just your own.

### Viewing Support Tickets

**From Support Tickets Dashboard**:
1. Go to **Support Tickets** from top menu
2. View all open tickets across all projects
3. See ticket statistics and status breakdown
4. Filter by status (Open, In Progress, Closed)

**From Project Detail Page**:
- Scroll to **Support / Service Tickets** section
- View all tickets for that specific project
- See ticket status and activity history

### Creating Support Tickets

**When to Create**:
- Customer reports an installation issue
- Customer requests service or maintenance
- Follow-up is needed on a customer concern
- Any operational issue needs tracking

**How to Create**:
1. Open the project (from Projects page or Support Tickets Dashboard)
2. Scroll to **Support / Service Tickets** section
3. Click **Create Ticket** button
4. Enter:
   - **Title**: Brief description of the issue (required)
   - **Description**: Detailed explanation (optional)
5. Click **Create**

**Ticket Number**:
- Each ticket gets a unique number (format: RE followed by 8 digits)
- Use this number when discussing with other teams

### Managing Ticket Status

**Ticket Statuses**:
- **Open**: Ticket just created, needs attention
- **In Progress**: Someone is working on the issue
- **Closed**: Issue resolved, ticket complete

**Updating Status**:
- Tickets start as "Open"
- Change to "In Progress" when you start working on it
- Change to "Closed" when issue is resolved

### Adding Follow-ups

**When to Add Follow-ups**:
- You have an update on the issue
- Customer provided more information
- You need to schedule a follow-up date
- You want to add notes about progress
- You've taken action on the ticket

**How to Add**:
1. Open the ticket (from project page or Support Tickets Dashboard)
2. Scroll to **Activities** section
3. Enter your note in the "Add Follow-up" form
4. Optionally set a **Follow-up Date** if action is needed later
5. Click **Add Follow-up**

**Follow-up Dates**:
- Set a date when you need to take action
- Helps you remember to follow up
- Dashboard shows tickets with past follow-up dates as overdue

### Closing Tickets

**When to Close**:
- Customer confirms issue is resolved
- Problem has been fixed
- Installation issue is complete
- No further action needed

**How to Close**:
1. Open the ticket
2. Click **Close Ticket** button
3. Ticket status changes to "Closed"
4. Add a final follow-up note explaining resolution (recommended)

**Important**: Once closed, only Administrators can add notes to tickets. Make sure all information is added before closing.

### Best Practices for Support Tickets

- **Create tickets immediately**: Don't delay when issues are reported
- **Use clear titles**: Make it easy to understand the issue
- **Add detailed descriptions**: Provide context and background
- **Update status promptly**: Keep ticket status current
- **Set follow-up dates**: Schedule when action is needed
- **Add follow-ups regularly**: Document all progress and actions
- **Close tickets promptly**: Mark resolved when customer confirms
- **Monitor overdue tickets**: Check for tickets with past follow-up dates

## Document Uploads

### Understanding Document Management

Documents are stored securely in the system and can be accessed by authorized users. You can upload documents for any project you have access to.

### Document Categories

**Photos / Videos**:
- Installation photos
- Site photos
- Video recordings
- Progress photos

**Documents**:
- Certificates
- Compliance documents
- Approval letters
- Registration documents
- Reports

**Sheets**:
- Spreadsheets
- Data files
- Calculation sheets
- Compliance sheets

### Uploading Documents

**How to Upload**:
1. Open the project detail page
2. Scroll to **Documents** section
3. Click **Upload File** button
4. Select file from your computer
5. Choose **Category** (Photos/Videos, Documents, or Sheets)
6. Enter **Description** (optional but recommended)
7. Click **Upload File**

**Supported File Types**:
- PDF documents
- Images (JPG, PNG, GIF, WebP, BMP)
- Videos (MP4, MPEG, QuickTime, AVI)
- Office documents (Word, Excel, PowerPoint)
- Text files (TXT, CSV)

**File Size Limits**:
- Check with administrator for current size limits
- Large files may take time to upload
- Ensure stable internet connection for large uploads

### Document Organization

**Best Practices**:
- **Use correct category**: Helps others find documents easily
- **Add descriptions**: Explain what the document is
- **Upload promptly**: Don't wait until project is complete
- **Name files clearly**: Use descriptive file names before uploading
- **Upload as you go**: Upload documents as you receive them

**What Documents to Upload**:
- Installation photos and videos
- Completion certificates
- Compliance documents
- DISCOM approval documents
- MNRE registration documents
- Net meter installation documents
- Any other project-related documents

### Viewing and Downloading Documents

**Viewing Documents**:
- All documents are listed in the Documents section
- Click on document name to view (if supported)
- View document details (category, description, upload date, uploaded by)

**Downloading Documents**:
- Click **Download** button next to document
- File downloads to your computer
- Use for sharing or backup purposes

### Document Management Tips

- **Upload immediately**: Don't accumulate documents
- **Use descriptions**: Help others understand document purpose
- **Organize by category**: Makes finding documents easier
- **Verify uploads**: Confirm documents uploaded successfully
- **Keep originals**: Maintain copies on your computer
