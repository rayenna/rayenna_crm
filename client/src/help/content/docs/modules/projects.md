# Projects Module

## Overview

The Projects module is the core of Rayenna CRM, managing the complete lifecycle of solar energy projects from initial lead to final completion. It serves as the central hub for coordinating sales, operations, finance, and customer service activities.

## Project Creation

### Who Can Create Projects

- **Sales users**: Can create projects for their assigned customers
- **Administrators**: Can create projects for any customer

### Creating a New Project

**Step 1: Access Projects Page**
1. Go to **Projects** page from the top menu
2. Click **New Project** button (top right)

**Step 2: Select Customer**

**Customer Selection**:
- Search for existing customer by name, ID, or consumer number
- Select customer from dropdown list
- Customer cannot be changed after project creation
- If customer doesn't exist, create customer first from Customer Master

**Required**: Customer must be selected before proceeding

**Step 3: Fill in Project Details**

**Customer & Project Details Section**:

- **Segment** (Required): 
  - Residential Subsidy
  - Residential Non-Subsidy
  - Commercial Industrial

- **Project Type** (Required):
  - EPC Project
  - Panel Cleaning
  - Maintenance
  - Repair
  - Consulting
  - Resale
  - Other Services

**Step 4: Sales & Commercial Information**

**Lead Source**:
- Website
- Referral (requires Referral Name)
- Google
- Channel Partner (requires Channel Partner Name)
- Digital Marketing
- Sales
- Management Connect
- Other (requires Other Details)

**Project Information**:
- **System Capacity (kW)**: Solar system capacity in kilowatts
- **Order Value (₹)**: Total project value/contract amount
- **Confirmation Date** (Required): Date when customer confirmed the order
- **Year (FY)**: Automatically calculated from Confirmation Date (Financial Year format: 2024-25)
- **Project Status**: Current status of the project (see Status Stages section)

**Availing Loan / Financing**:
- **Availing Loan/Financing?**: Check **Yes** if the customer is availing loan or financing for the project; leave unchecked for No.
- If **Yes**, **Financing Bank** (required): Select from dropdown (e.g. State Bank of India, HDFC Bank, ICICI Bank, Other). If you select **Other**, **Other Bank Name** (required): Enter the bank name (alphanumeric, spaces and basic punctuation allowed).

**Technical Details**:
- **Roof Type**: Concrete-Flat, Concrete-Sloped, Tile, Thatched, Asbestos, Metal, Others
- **System Type**: Off-Grid, On-Grid, Hybrid

**Lost Status Fields** (if Project Status is "Lost"):
- **Lost Date** (Required): Date when project was lost
- **Reason for Loss** (Required): 
  - Lost to Competition
  - No Budget
  - Indefinitely Delayed
  - Other (requires Other Reason Details)

**Step 5: Save Project**
1. Review all information for accuracy
2. Click **Create** button
3. Project is created and assigned a unique Project Number (SL No)
4. You'll see a success message
5. You'll be redirected to Projects list

### Project Number Generation

- Project Number (SL No) is automatically generated by the system
- Format: Sequential number
- Cannot be changed after creation
- Used to uniquely identify projects

### Required Fields for New Projects

**Minimum Required**:
- Customer (must be selected)
- Segment (Project Type)
- Project Type (Service Type)
- Confirmation Date

**Conditional Requirements**:
- Lost Date and Reason for Loss (if status is Lost)
- Referral Name (if Lead Source is Referral)
- Channel Partner Name (if Lead Source is Channel Partner)
- Other Details (if Lead Source is Other)
- Other Reason Details (if Lost Reason is Other)
- Financing Bank (if Availing Loan/Financing is Yes)
- Other Bank Name (if Financing Bank is Other)

### Project Creation Best Practices

**Complete Information**:
- Fill in all available fields at creation
- Add system capacity and order value
- Set accurate confirmation date
- Select appropriate lead source

**Accuracy**:
- Verify customer information before creating
- Double-check financial figures
- Ensure dates are correct
- Validate technical specifications

**Organization**:
- Use consistent naming conventions
- Keep project information standardized
- Maintain accurate status updates
- Update information regularly

## Financial Tracking

### Understanding Financial Tracking

Financial tracking in projects enables comprehensive monitoring of project value, payments received, outstanding balances, and profitability. This information is critical for cash flow management and business analysis.

### Payment Tracking

**Who Can Edit Payments**:
- **Finance users**: Can update all payment information
- **Administrators**: Full access to payment tracking

**Payment Milestones**:

1. **Advance Received**:
   - Amount (₹)
   - Date received
   - First payment from customer

2. **Payment 1**:
   - Amount (₹)
   - Date received
   - Second milestone payment

3. **Payment 2**:
   - Amount (₹)
   - Date received
   - Third milestone payment

4. **Payment 3**:
   - Amount (₹)
   - Date received
   - Fourth milestone payment

5. **Last Payment**:
   - Amount (₹)
   - Date received
   - Final payment

**Payment Validation Rules**:
- Amount and Date must both be provided or both be empty
- Cannot enter amount without date
- Cannot enter date without amount
- System validates payment pairs automatically

### Automatic Calculations

**Total Amount Received**:
- Automatically calculated from all payment milestones
- Sum of: Advance + Payment 1 + Payment 2 + Payment 3 + Last Payment
- Updated in real-time as payments are entered

**Balance Amount**:
- Automatically calculated: Order Value - Total Amount Received
- Shows outstanding amount to be collected
- Updates automatically when payments are added

**Payment Status**:
- **PENDING**: No payments received
- **PARTIAL**: Some payments received but not fully paid
- **FULLY_PAID**: Total amount received equals or exceeds order value
- Automatically updated based on payment amounts

### Financial Information Display

**From Project Detail Page**:
- Payment Status badge (color-coded)
- Total Amount Received
- Balance Amount
- Individual payment details with dates
- Order Value (Project Cost)

**Financial Metrics**:
- Project profitability
- Gross profit calculation
- Expected profit (if available)
- Total project cost (actual costs incurred)

### Profitability Tracking

**Gross Profit Calculation**:
- Gross Profit = Order Value - Total Project Cost
- Calculated automatically when both values are available
- Displayed on project detail page

**Expected Profit**:
- Estimated profit based on project value
- Calculated using margin percentage
- Used for planning and forecasting

**Profitability Percentage**:
- (Gross Profit / Order Value) × 100
- Shows profit margin as percentage
- Useful for performance analysis

### Financial Best Practices

**Regular Updates**:
- Update payments as soon as received
- Enter payment dates accurately
- Keep payment information current
- Verify amounts before saving

**Accuracy**:
- Double-check payment amounts
- Ensure dates are correct
- Verify total calculations
- Maintain payment records

**Tracking**:
- Monitor payment status regularly
- Track outstanding balances
- Follow up on pending payments
- Update payment milestones promptly

## Status Stages

### Understanding Project Status

Project status tracks the current stage of a project through its lifecycle. Each status represents a specific phase, from initial inquiry to final completion or loss.

### Project Status Stages

**1. Lead**
- **Description**: Initial inquiry or potential customer
- **Characteristics**: 
  - Customer has shown interest
  - No commitment yet
  - Initial contact stage
- **Typical Actions**: 
  - Contact customer
  - Gather requirements
  - Schedule site survey

**2. Site Survey**
- **Description**: Site visit and assessment completed
- **Characteristics**:
  - Site has been visited
  - Technical assessment done
  - Feasibility evaluated
- **Typical Actions**:
  - Document site details
  - Prepare proposal
  - Move to proposal stage

**3. Proposal**
- **Description**: Proposal sent to customer
- **Characteristics**:
  - Proposal document prepared
  - Sent to customer for review
  - Awaiting customer decision
- **Typical Actions**:
  - Generate proposal (AI-assisted)
  - Follow up with customer
  - Address customer queries

**4. Confirmed Order**
- **Description**: Customer has confirmed the order
- **Characteristics**:
  - Customer has committed
  - Order confirmed
  - Project can proceed
- **Typical Actions**:
  - Set confirmation date
  - Begin project planning
  - Start execution activities

**5. Installation**
- **Description**: Installation in progress
- **Characteristics**:
  - Installation work started
  - Equipment being installed
  - Active execution phase
- **Typical Actions**:
  - Track installation progress
  - Update milestone dates
  - Monitor completion

**6. Submitted for Subsidy**
- **Description**: Project submitted for subsidy processing
- **Characteristics**:
  - Installation completed
  - Subsidy application submitted
  - Awaiting subsidy approval
- **Typical Actions**:
  - Track subsidy status
  - Follow up on approval
  - Update subsidy dates

**7. Completed**
- **Description**: Project completed successfully
- **Characteristics**:
  - Installation complete
  - System operational
  - Project finished
- **Typical Actions**:
  - Final documentation
  - Customer handover
  - Close project

**8. Completed - Subsidy Credited**
- **Description**: Project completed and subsidy received
- **Characteristics**:
  - Installation complete
  - Subsidy credited to account
  - Project fully closed
- **Typical Actions**:
  - Final accounting
  - Project closure
  - Archive documentation

**9. Lost**
- **Description**: Project did not proceed
- **Characteristics**:
  - Customer did not proceed
  - Project cancelled
  - No longer active
- **Required Information**:
  - Lost Date (required)
  - Reason for Loss (required)
  - Other Reason Details (if reason is "Other")
- **Restrictions**:
  - Cannot be edited after marking as Lost
  - Only Admin can delete Lost projects
  - Order Value and Total Project Cost set to 0

### Status Progression

**Normal Flow**:
Lead → Site Survey → Proposal → Confirmed Order → Installation → Submitted for Subsidy → Completed → Completed - Subsidy Credited

**Alternative Flow**:
Any stage → Lost (if project doesn't proceed)

**Status Updates**:
- Update status as project progresses
- Status changes reflect current project stage
- Status history is maintained
- Cannot revert from Lost status

### Status-Based Permissions

**Sales Users**:
- Can update status up to Confirmed Order
- Can mark projects as Lost (for their projects)
- Cannot edit Lost projects

**Operations Users**:
- Can update status from Installation onwards
- Can update execution milestones
- Can mark projects as Lost

**Finance Users**:
- Can view all statuses
- Cannot change project status
- Can update payment information

**Administrators**:
- Can update any status
- Can delete Lost projects
- Full access to all status changes

### Status Best Practices

**Regular Updates**:
- Update status promptly as project progresses
- Keep status current and accurate
- Reflect actual project stage
- Don't skip status stages

**Accuracy**:
- Verify project stage before updating
- Ensure status matches reality
- Update related dates when changing status
- Document status changes

**Communication**:
- Notify team when status changes
- Update stakeholders on progress
- Document reasons for status changes
- Maintain status history

## Sales and Operations Collaboration

### Understanding Collaboration

The Projects module enables seamless collaboration between Sales and Operations teams. Each team has specific responsibilities and permissions, ensuring efficient project execution while maintaining data integrity.

### Sales Team Responsibilities

**Project Creation**:
- Create new projects
- Select and link customers
- Enter initial project information
- Set lead source and commercial details

**Sales & Commercial Section** (Sales and Admin can edit; Finance, Operations, and Management can view only):
- **Lead Source**: Track where the lead came from
- **Availing Loan/Financing**: Whether customer is availing loan; if Yes, **Financing Bank** (and **Other Bank Name** when Other) is required
- **System Capacity**: Enter solar system capacity
- **Order Value**: Set project contract value
- **Confirmation Date**: Record when customer confirmed
- **Project Status**: Update status up to Confirmed Order
- **Roof Type**: Document roof characteristics
- **System Type**: Specify grid type (On-Grid, Off-Grid, Hybrid)

**Project Management**:
- Update project status (Lead → Site Survey → Proposal → Confirmed)
- Generate AI proposals for customers
- Add remarks and internal notes
- Create support tickets for customer issues
- Upload sales-related documents

**Customer Communication**:
- Maintain customer relationships
- Address customer queries
- Follow up on proposals
- Track customer interactions

### Operations Team Responsibilities

**Project Execution** (Operations and Admin can edit):

  - **Completion Report Submission Date**: Date completion report was submitted
  - **Net Meter Installation Date**: Date net meter was installed
  - **Total Project Cost**: Actual costs incurred in the project
  - **Panel Brand**: Brand of solar panels used
  - **Inverter Brand**: Brand of inverters used

**Status Management**:
- Update status from Installation onwards
- Track installation progress
- Monitor completion milestones
- Update execution dates

**Documentation**:
- Upload installation photos and videos
- Upload compliance documents
- Upload completion reports
- Organize project artifacts

**Compliance Tracking**:
- Track MNRE registration
- Monitor DISCOM approvals
- Follow up on subsidy applications
- Update compliance dates

### Finance Team Responsibilities

**Payment Tracking** (Finance and Admin can edit):
- **Payment Milestones**:
  - Advance Received (Amount and Date)
  - Payment 1 (Amount and Date)
  - Payment 2 (Amount and Date)
  - Payment 3 (Amount and Date)
  - Last Payment (Amount and Date)

**Financial Management**:
- Update payment information
- Track payment status
- Monitor outstanding balances
- Export financial data to Tally

**View-Only Access**:
- View all project information
- View execution milestones
- View documents and artifacts
- Cannot edit project details (except payments)

### Collaboration Workflow

**Project Initiation** (Sales):
1. Sales creates project with customer information
2. Enters commercial details and order value
3. Sets confirmation date and initial status
4. Generates proposal if needed

**Project Execution** (Operations):
1. Operations takes over when status is Installation
2. Updates execution milestones
3. Tracks compliance dates
4. Uploads installation documents
5. Updates installation completion date

**Payment Processing** (Finance):
1. Finance updates payments as received
2. Tracks payment milestones
3. Monitors outstanding balances
4. Updates payment status

**Project Completion** (Operations + Finance):
1. Operations marks installation complete
2. Operations submits completion reports
3. Finance tracks final payments
4. Operations updates subsidy status
5. Project marked as Completed

## Linked Support Tickets and Documents

### Understanding Linked Resources

Projects can have multiple support tickets and documents linked to them. These resources provide comprehensive project documentation and enable efficient customer service management.

### Support Tickets

**What Are Support Tickets**:
- Customer service requests linked to projects
- Track customer issues and resolutions
- Manage follow-ups and activities
- Monitor support ticket status

**Who Can Create Tickets**:
- **Sales users**: Can create tickets for their projects
- **Operations users**: Can create tickets for any project
- **Administrators**: Can create tickets for any project

**Creating Support Tickets**:
1. Go to Project Detail page
2. Scroll to "Support / Service Tickets" section
3. Click **Create Ticket** button
4. Enter ticket information:
   - **Title**: Brief description of the issue
   - **Description**: Detailed description (optional)
5. Click **Create**
6. Ticket is created with unique ticket number (RE########)

**Ticket Information**:
- **Ticket Number**: Unique identifier (RE + 8 digits)
- **Title**: Issue description
- **Status**: OPEN, IN_PROGRESS, CLOSED
- **Created Date**: When ticket was created
- **Created By**: User who created the ticket
- **Closed Date**: When ticket was closed (if closed)

**Managing Tickets**:
- **View Ticket**: Click on ticket to see details
- **Add Follow-up**: Add activity notes with follow-up dates
- **Close Ticket**: Mark ticket as closed when resolved
- **Delete Ticket**: Admin only - permanently delete ticket

**Ticket Activities**:
- **Follow-up Notes**: Add notes about ticket progress
- **Follow-up Date**: Set date for next follow-up
- **Activity Timeline**: View all activities chronologically
- **Created By**: Track who added each activity

**Ticket Status**:
- **OPEN**: New ticket, not yet addressed
- **IN_PROGRESS**: Ticket is being worked on
- **CLOSED**: Ticket resolved and closed

**Viewing Tickets**:
- From Project Detail page: See all tickets for the project
- From Support Tickets Dashboard: View all tickets across projects
- Filter by status: Open, In Progress, Closed
- Search by ticket number or title

### Documents and Artifacts

**What Are Documents**:
- Files uploaded to projects
- Photos, videos, PDFs, spreadsheets
- Project documentation and compliance files
- Organized by category

**Document Categories**:
- **Photos / Videos**: Installation photos, site videos, progress images
- **Documents**: PDFs, proposals, contracts, compliance documents
- **Sheets**: Excel files, data sheets, calculations

**Who Can Upload Documents**:
- **Sales users**: Can upload documents to their projects
- **Operations users**: Can upload documents to any project
- **Administrators**: Can upload documents to any project

**Uploading Documents**:
1. Go to Project Detail page
2. Click **Edit** button
3. Scroll to "File Uploads" section
4. Click **Choose File** and select file
5. Select **Category** (required)
6. Enter **Description** (optional)
7. Click **Upload File**
8. File is uploaded and linked to project

**File Requirements**:
- **Supported Types**: Images (JPEG, PNG, GIF), PDFs, Office documents (Excel, Word)
- **Maximum Size**: 10MB per file
- **Validation**: System validates file type and size

**Viewing Documents**:
- From Project Detail page: See all documents in "Key Artifacts" section
- View document: Click view button to open file
- Download document: Click download button to save file
- Document information: See category, upload date, uploaded by

**Document Permissions**:
- **View**: All users can view documents (with role permissions)
- **Download**: All users can download documents (with role permissions)
- **Delete**: 
  - Admin can delete any document
  - Uploader can delete their own documents
  - Exception: AI Generated Proposal PDFs can only be deleted by Admin

**Document Organization**:
- Documents grouped by category
- Displayed in grid layout
- Shows file name, category, uploader
- Easy to find and access

## Project Filtering and Search

### Search Functionality

**Search Bar**:
- Located at top of Projects page
- Searches across multiple fields:
  - Customer name
  - Customer ID
  - Consumer number
- Real-time search with debounce
- Case-insensitive search

**Search Tips**:
- Use partial matches (e.g., "solar" finds "Solar Energy")
- Search by customer ID for quick lookup
- Use consumer number for utility-related searches
- Clear search to see all projects

### Filtering Options

**Status Filter**:
- Filter by project status
- Multiple statuses can be selected
- Options: Lead, Site Survey, Proposal, Confirmed, Installation, Completed, etc.
- Shows projects in selected statuses

**Type Filter**:
- Filter by project segment
- Options: Residential Subsidy, Residential Non-Subsidy, Commercial Industrial
- Multiple types can be selected

**Service Type Filter**:
- Filter by project service type
- Options: EPC Project, Panel Cleaning, Maintenance, Repair, Consulting, Resale, Other Services
- Multiple service types can be selected

**Salesperson Filter** (non-Sales users):
- Filter by assigned salesperson
- Select one or more salespersons
- Shows projects for selected salespersons
- Leave empty to see all projects

**Availing Loan Filter**:
- Checkbox: **Availing Loan**
- When checked, shows only projects where Availing Loan/Financing is Yes (and a bank is selected)
- Useful to see all projects tied to financing/loan

**Support Ticket Status Filter**:
- Filter projects by support ticket status
- Options:
  - Has Tickets: Projects with any tickets
  - Open: Projects with open tickets
  - In Progress: Projects with in-progress tickets
  - Closed: Projects with closed tickets
  - No Tickets: Projects without tickets
- Multiple statuses can be selected

**Filter Behavior**:
- Filters apply immediately
- Can combine multiple filters
- Can combine with search
- Pagination resets when filters change
- Clear filters to see all projects

### Sorting Options

**Default Sort**:
- Projects sorted by creation date (newest first)
- Can be changed using sort options

**Sort Options**:
- By creation date
- By confirmation date
- By project value
- By customer name
- By status

## Troubleshooting

### Cannot Create Project

**Issue**: "New Project" button not visible or creation fails

**Solutions**:
- Verify you have Sales or Admin role
- Check required fields are filled
- Ensure customer is selected
- Verify confirmation date is entered
- Contact administrator if issue persists

### Cannot Edit Project

**Issue**: Edit button not visible or edit fails

**Solutions**:
- Check your user role permissions
- For Sales users: Ensure project is assigned to you
- Verify project is not in Lost status
- Check section-specific permissions
- Refresh page and try again
- Contact administrator if needed

### Payment Not Saving

**Issue**: Payment information not saving

**Solutions**:
- Verify you have Finance or Admin role
- Check both amount and date are entered
- Ensure payment section is accessible
- Verify form validation errors
- Refresh page and try again

### Document Upload Failing

**Issue**: Document upload fails or file not visible

**Solutions**:
- Check file type is supported
- Verify file size is under 10MB
- Ensure category is selected
- Check you have upload permissions
- Try uploading again
- Contact administrator if issue persists

### Support Ticket Not Creating

**Issue**: Cannot create support ticket

**Solutions**:
- Verify you have Sales, Operations, or Admin role
- Check ticket title is entered
- Ensure project exists and is accessible
- Refresh page and try again
- Contact administrator if needed
